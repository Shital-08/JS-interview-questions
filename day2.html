1. What is creation phase and execution phase?

     Creation Phase : The JS engine invokes a function but does not start its execution during the creation phase. 
     The JS engine starts the compilation step in this phase and examines the function code for compiling but does not run it.
     During the creation phase, two-state components are created:
     1. Lexical Environment
     2. Variable Environment

     Execution : The JS engines scan over the function in the code one more time, updating the variable object with the values of the variables and then running the code, which is known as an execution phase.
     Variable binding initializations, variable assignments, mutability and immutability checking, variable binding deletions, function call execution, and so on are all occurring during this phase.

2. What are objects in javascript?

JavaScript object is a non-primitive data-type that allows you to store multiple collections of data in the form of key-value pairs.

3. What are function constructors?

The Function() constructor creates a new Function object. Calling the constructor directly can create functions dynamically.

4. Explain prototypes.

A prototype is an existing inbuilt functionality in JavaScript. Whenever we create a JavaScript function, JavaScript adds a prototype property to that function. 
         A prototype is an object, where it can add new variables and methods to the existing object.

5. What is prototype chain.

The prototype is itself an object, so the prototype will have its own prototype, making what's called a prototype chain. 
         The chain ends when we reach a prototype that has null for its own prototype.

6. Give an example of inheritance using function constructor.

Inheritance is something when child is inheriting the properties of parent.

7. What are callbacks?

A callback function is a function passed into another function as an argument, which is then invoked inside the outer function.
